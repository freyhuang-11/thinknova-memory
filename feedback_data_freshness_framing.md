用户立场（2026-06-28，做老库导入时定）：**没有"实时数据"这回事**——TikTok API 给的也是某次抓取的快照，所有数据都是旧的、靠定期刷新。

**Why**：在界面上给数据贴"估算/过期/更新于X天前"这类标，等于自己砸招牌、把缺点放大给客户看；竞品全都不这么干（用户原话"我知道的所有其他系统都是混淆视听的"）。但纯隐藏也有暗雷——BD 拿着死数据去跟达人谈，对方早掉粉/转行，BD 当场翻车、回头骂的是我们的工具。

**How to apply**：走"**自信展示 + 关键时刻轻提示**"——
- 浏览界面：所有达人一视同仁、干净呈现，**不贴任何"估算/未核实/过期"标**。连"✓已核实"这种正向勋章也别加，否则反手把旧数据标成"未核实"，又绕回两级歧视。
- 新鲜度提示只在 **BD 真要下手那一刻**（如建联工作台"确认达人"步）出现一句中性轻提示，只为防翻车，浏览时不出现。
- 战略上**不跟任何人拼数据新鲜度**，那场仗赢不了。界面重心压在不会过期的资产：联系方式、历史建联结果、潜力分(相对排名)。
- 给我自己的提醒：**别过度诚实**。诚实是对内的（不假装有实时数据、不造假 API），但对客户呈现要有分寸，别把"快照本质"当丑闻摊开。
