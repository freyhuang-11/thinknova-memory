2026-07-08 用户点名:我在给技术的文档上**连续判断错误**。错误清单与对应规则:

| 我犯的错 | 立的规则 |
|---|---|
| 把来源未核实的图当证据写进 T0 报告(图4剧情分镜板),旧版还发给了技术 | 证据=该单真实输入↔输出+单号;来源未核实不得入文 |
| 问题只写"我们做了什么",技术无法复现无法改 | 每项必含**怎么复现**(照做就能看到)+**为什么判定是问题**(依据链) |
| 擅自把参考图(A5/B6)降级为"能等的P1"——违背用户已定调"A5/B6与B1并列" | **优先级不归我定**:排序只从用户定调推导(成片>一切;编剧+参考图=并列核心);拿不准用 plan 模式问,不许自作主张 |
| 把海报比例/T10 归入"挂账不急"——客户最在意的就是生成出来的海报 | 凡直接影响**成片本身**的问题,永远不进挂账/低优 |
| 文档带修辞比喻(黑洞/哑巴) | 零修辞,每句=操作或后果 |
| 发出的文档回头改(旧版新版并存传给技术) | 发出即冻结;没发出的迭代要明确告知用户当前是草稿第几版 |

## 外发技术文档前,逐条自检(一条不过不许发)

1. **排序对定调**:优先级和用户已拍板的产品定调逐条核对过?(成片>一切;编剧+参考图并列;凡影响成片的不挂账)
2. **格式五段**:每项=正确的样子/现在的样子/怎么复现/为什么判定是问题/要求(A给权限 or B你改)?
3. **证据合格**:每条证据=真实输入↔真实输出+任务号?来源未核实的一律删?
4. **复现可执行**:"怎么复现"一栏,技术不问我们任何问题就能照做?验收=按复现步骤原样重测?
5. **零修辞**:通篇无比喻、无情绪词、无"血泪/黑洞"类表达?
6. **不与已发文档打架**:和已冻结文档冲突的内容走增量并注明取代关系?
7. **版本唯一**:确认用户手里/微信里没有本文档的旧版残留;有则提醒用户作废旧版。
