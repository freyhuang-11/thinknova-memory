系统提示只提供**日期**,不提供**时刻**。所以我天然不知道现在几点,也感知不到对话之间过了多久。

**不许凭感觉说时间。** 需要时刻时执行:

```bash
date "+%Y-%m-%d %H:%M:%S %A"
```

**Why**:2026-07-22 03:08,用户指出"你分不清楚时间,我每次和你聊都是过时间的"。此前我在回复里写"现在是凌晨3点多"——那是猜的,恰好蒙对。更实际的危害:核验任务时靠 `created_at` 判断哪单是刚烧的、一个 processing 的单是刚提交还是卡住了,不知道当前时刻就只能盲猜。

**How to apply**:
1. 判断"刚才/现在/多久之前"之前,先 `date`;
2. 核验烧单时,用当前时间和任务 `created_at` 相减,才能说清"这单跑了多久""这是不是最新一单";
3. 跨天工作时(用户常做到凌晨),日期会翻篇,别默认还是昨天;
4. 记录事故/结论进 vault 或记忆时,时间戳用 `date` 取,不要写"今晚""刚才"这类相对说法——[[feedback-memory-keep-current]] 要求记当前状态,相对时间过几天就失效了。
