# Compass 真正的核心方向（2026-06-28 大方向已定，具体执行未定）

**一句话**：建联（outreach）只是**地基，已经够用**；真正值钱、要死磕的是两根柱子——
1. **建联前分析（选对人）**——尽可能做到我们能做到的极致；
2. **建联后管理（管到出单）**——投流决策、深度跟踪、数据分析。
> 这覆盖掉早期"对标 Proboost 把建联做漂亮"的旧定位。**纯建联没有意义**（用户原话）。

## 竞品验证结论（别再重复踩）
- **市场分两派**：①**数据分析派** Kalodata(2亿画像/500天)、EchoTik(新加坡、营销称10亿其实灌水、数据被吐槽不准、明说"找到后无执行工具")、蝉妈妈类——**核心是选品/数据，不做执行**。②**建联/执行派** Cruva、Reacher（美区、全漏斗强对手）、达连/达秘/Proboost（中国跨境、速度+基础CRM）、花漾（全链路、1000万库、6年）。
- **Compass 属执行派**，**别去跟数据派拼数据库**（那是灌水+不准+追不上的沟）。执行派护城河是 workflow/管理，**小、可竞争**。
- **"10亿达人"是营销**：TikTok 没那么多创作者；那是画像/记录覆盖量，且准确性存疑。**不要把竞品营销数当事实。**

## TikTok 原生已给（站肩上，别重造）
- **达人履约率 CFR**（寄样后会不会发的概率）、**达人样品分4维**（履约/勤奋/出单/内容质量）、**Manage Creators**（近90天GMV/产出/打标签）、**GMV Max**（算法自动投流）。

## 我们的差异化"刀"（组合，不是单点）
- **建联前**：在 TikTok 通用分之上，加**"针对这个商家商品"的产品级契合度**（扩展抓达人近30天内容有没有本品类货+评论率+购买意向）+ **SEA 跨语言** + 自有**回复率/履约率**（越用越准）。TikTok/Reacher 的分是通用的，给不了"契合你的商品"。
- **建联后**：**投流决策支持**＝每条达人视频拉早期信号(点击率/评论速度/完播/自然出单)对标基准(CTR 2–3%、RPM $25–60)+两笔成本(广告费+佣金) → 给"值得投/再看/不投"。**这是没人做成干净决策工具的真空**（Reacher凑量、Cruva事后测、GMV Max黑箱）。+ **出单/ROI归因**(联盟订单API) + **复投/防流失**(开放合作58%三月流失没人管)。

## 诚实边界（判断时必记）
- **脚本/服务器爬不行**（扛不住持续看每个达人最近内容）→ 用 **Chrome 扩展**（BD 已登录会话内读前端、限速、灰区、随TikTok改版长期维护）。竞品就是扩展。
- **GMV Max 在自动化投流本身** → 我们做"投前决策支持 + 投后关系/履约管理"，**不做"替你自动投"**（打不过平台）。
- **CFR/样品分能否走 API 拿、还是只在后台 UI** = 待验证；拿不到就扩展抓。
- **护城河(自有回复/履约/出单数据 + 大达人库)要时间**，不是上线即有；冷启动靠 SG/MY 现成数据当楔子。
- **Cruva/Reacher 是真对手**，赢点是上面几刀的组合 + SEA，不是某个单一杀手功能。

## 现状 / 紧迫性
- 已做：**地基建联 + 建联后管理的 UI 壳**（会话/寄样/合作有 UI，深度功能未接）。**两根柱子的核心功能都还没开始做。**
- **具体执行未定**；时间紧，系统做出来**还要做市场验证**。优先级应是：建联前评分 / 投流决策卡 / 复投管理 → 最前；纯建联美化 → 往后。
